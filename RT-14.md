# RT-14 — Preparação para CI/CD e Docker Hub

Este relatório documenta a prontidão da **RSD-Stack** para integração em pipelines de CI/CD e publicação em registros de imagens (Docker Hub), garantindo escalabilidade e controle de versão.

## 1. Padronização de Tags

O sistema de tags foi normalizado para seguir o padrão `rsd/<serviço>:<versão>`. 
- O arquivo `docker-compose.yml` agora utiliza a variável de ambiente `${RSD_VERSION}` com fallback para `v1.0.0`.
- Isso permite que o pipeline de CI defina a versão dinamicamente durante o build e o deploy.

## 2. Automação de Build (CI-Ready)

O script `ops/build-all.sh` foi aprimorado para aceitar argumentos de versão:
```bash
# Exemplo de uso em CI/CD
bash ops/build-all.sh v1.2.3
```
Esta funcionalidade garante que todas as imagens da stack sejam construídas com a mesma tag de versão, mantendo a consistência do artefato final.

## 3. Gates de Integridade e Governança

A stack agora possui gates automatizados que garantem a conformidade antes de qualquer publicação:
- **Integridade Criptográfica:** Validação de hash SHA256 do `docker-compose.yml` (RT-12).
- **Governança de Recursos (RT-11A):** O pipeline de CI é instruído a falhar caso algum serviço seja definido sem limites de CPU, garantindo que apenas containers "bem comportados" cheguem à produção.
- **Hardening de Origem:** As validações arquiteturais (RT-09) garantem que apenas imagens seguras e em conformidade sejam geradas para upload.

## 4. Status Final
- **Prontidão CI/CD:** ALTA (com gates de integridade, reprodutibilidade e governança de recursos).
- **Padronização de Tags:** CONCLUÍDA.
- **Portabilidade:** 100% Linux-compliant.
