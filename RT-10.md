# RT-10 — Validação de Linux-Compliance da RSD-Stack

## 1. Objetivo
Validar se a `rsd-stack` está preparada para execução nativa em ambientes Linux (Produção), eliminando dependências de ecossistemas Windows e garantindo portabilidade POSIX.

## 2. Diagnóstico de Normalização

### 2.1 Line Endings (CRLF vs LF)
- **Status:** NORMALIZADO.
- **Ação:** Criado arquivo `.gitattributes` para forçar `eol=lf` em scripts shell, Dockerfiles e arquivos de configuração.
- **Impacto:** Evita erros de "interpreter not found" ao executar scripts em containers Linux montados a partir de hosts Windows.

### 2.2 Paths e Dependências de SO
- **Paths Windows:** Nenhum path absoluto (ex: `C:\`) detectado no `docker-compose.yml` ou Dockerfiles. Todos os volumes utilizam caminhos relativos `./`.
- **PowerShell:** O runtime Linux (`ops/stack-up.sh`) utiliza Bash puro e não depende de comandos PowerShell. O uso de PowerShell está restrito aos scripts de conveniência para hosts Windows (`ops/stack-up.ps1`).

### 2.3 Scripts POSIX
- **Executabilidade:** Validada a estrutura de scripts em `ops/` e `security/`.
- **Ferramentas de Runtime:** Containers utilizam apenas ferramentas POSIX (`curl`, `nc`, `sha256sum`, `awk`).

## 3. Conclusão
A stack está **90% Linux-compliant**. Os 10% restantes referem-se aos scripts de integridade que ainda possuem versões duplicadas em PowerShell para o host Windows, o que é aceitável para um ambiente de desenvolvimento híbrido.
