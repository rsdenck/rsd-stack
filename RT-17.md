# RT-17 — Release GitHub e Governança de Versão

Este relatório documenta a preparação para a criação de releases oficiais no GitHub e a governança de versionamento da **RSD-Stack**.

## 1. Estratégia de Versionamento

A stack adota o **Versionamento Semântico (SemVer)**:
- **v1.0.0**: Release estável inicial.
- **Push de Tags**: O pipeline de CI/CD é disparado apenas em tags que seguem o padrão `v*`.

## 2. Preparação do Repositório

| Item | Status | Observação |
| :--- | :--- | :--- |
| **Branch `main`** | ✅ Estável | Código base validado e pronto para merge. |
| **`.gitattributes`** | ✅ Presente | Garante normalização de line endings (LF). |
| **Scripts Linux** | ✅ Executáveis | Scripts em `ops/` e `security/` validados para POSIX. |
| **CI/CD Workflow** | ✅ Funcional | `docker-build.yml` configurado para Docker Hub. |

## 3. Governança de Artefatos

Cada release deve conter os seguintes artefatos versionados:
- **`docker-compose.yml`**: Orquestração principal.
- **`security/trust/compose.sha256`**: Hash de integridade do compose.
- **`security/trust/images.sha256`**: Registro de hashes das imagens publicadas.
- **`Dockerfile`**: Todas as imagens fixadas por SHA256 Digest (Parent/Builder).

## 4. Processo de Release (Checklist)

1. [ ] Garantir que `ops/validate-build.sh` passa localmente.
2. [ ] Atualizar `RSD_VERSION` no `.env.example`.
3. [ ] Commitar mudanças e criar tag: `git tag -a v1.0.0 -m "Release v1.0.0"`.
4. [ ] Push da tag: `git push origin v1.0.0`.
5. [ ] Validar a conclusão do workflow `RSD-STACK-DOCKERHUB-PUBLISH`.
6. [ ] Criar a release no GitHub vinculando a tag e anexando os RTs.

## 5. Conclusão

A infraestrutura de release está **PRONTA**. A stack agora possui um fluxo de publicação seguro, auditável e resistente a regressões.
