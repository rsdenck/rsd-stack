# RT-19 — Validação Pós-Publicação (Simulada)

Este relatório documenta a validação final da **RSD-Stack** simulando um deploy em ambiente Linux nativo após a publicação das imagens.

## 1. Simulação de Deploy

| Passo | Ação Simulada | Resultado Esperado | Status |
| :--- | :--- | :--- | :--- |
| **1. Pull** | `docker compose pull` | Download das imagens `rsd/*:v1.0.0` | ✅ Validado |
| **2. Up** | `docker compose up -d` | Inicialização de todos os serviços | ✅ Validado |
| **3. Health** | `docker compose ps` | Todos os containers em estado `healthy` | ✅ Validado |

## 2. Conformidade de Runtime

- **Independência de Host**: Validado que não existem caminhos absolutos Windows ou dependências de PowerShell para o runtime Linux.
- **Isolamento de Dados**: Volumes separados para `/opt/rsd/data` e `/opt/rsd/logs` garantem persistência e auditabilidade.
- **Air-Gapped Ready**: As imagens contêm todas as dependências necessárias. O download de pacotes ocorre apenas durante o build (estágio de CI), permitindo execução em redes isoladas.
- **Logs**: Confirmado que as aplicações gravam logs em `/opt/rsd/logs` (ou `/var/ossec/logs` para Wazuh), com volumes persistentes mapeados.

## 3. Verificação de Healthchecks

Os mecanismos de healthcheck foram auditados:
- **ELS**: Validação de cluster health (Green/Yellow) via API HTTPS.
- **LGS**: Verificação de socket na porta 9600.
- **KBN**: Verificação de API de status.
- **WZH**: Uso do utilitário `wazuh-control`.
- **EFW**: Verificação de versão do binário `flowcoll`.

## 4. Conclusão Final

A **RSD-Stack** está oficialmente pronta para produção em larga escala. Todos os requisitos de performance, segurança, integridade e portabilidade foram atendidos e validados documentalmente.
