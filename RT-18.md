# RT-18 — Publicação Docker Hub e Integridade de Imagens

Este relatório documenta a estratégia e a validação da publicação das imagens da **RSD-Stack** no Docker Hub, garantindo que apenas artefatos seguros e versionados sejam disponibilizados.

## 1. Padronização de Tags e Namespace

As imagens são publicadas sob o namespace oficial `rsd` seguindo o padrão:
- `rsd/els:<versão>`
- `rsd/lgs:<versão>`
- `rsd/kbn:<versão>`
- `rsd/wzh:<versão>`
- `rsd/efw:<versão>`

**Imutabilidade:** O pipeline de CI/CD está configurado para não permitir a sobrescrita de tags de versão já publicadas.

## 2. Metadados e Documentação (Docker Hub)

Para cada imagem, foi preparado um README específico (armazenado em `docker/hub_readmes/`) contendo:
- Descrição clara do serviço.
- Link para o repositório GitHub oficial.
- Aviso explícito de que a imagem faz parte da RSD-Stack e não suporta uso isolado.

## 3. Garantia de Build de Produção

As imagens publicadas no Docker Hub são garantidas como:
- **Build Linux Nativo**: Construídas em runners Linux (GitHub Actions).
- **Hardened**: Seguindo todos os controles do RT-09.
- **Resource Aware**: Validadas pelo gate de limites de CPU (RT-11A).
